在  src/main.cpp 这个程序中，有一些迭代过程中的数据十分重要，例如模拟退火中不同算子的接受率，迭代过程中最优解的L下降次数，分别在那个环节下降的，等等，这些数据对分析算法性能和边界条件是否遗漏有非常大的帮助

要求：
1) logger 尽量少改动原始代码，使用面向对象来设计，可以为 main 函数中每一个函数设计一个 log 函数，在函数内部使用极简的调用方式就能记录 log 信息，logger 在 src/logger.h 当中实现
2) log 信息要输出到指定的文件当中，且要写一个描述 log 文件结构的 log_format.md 文件到 docs/spec
3) logger 记录信息必须包括 
    - 模拟退火中不同算子的接受率
    - 迭代过程中最优解的L的全局下降曲线，并在全局上划分为多个环节
    - 最关键的环节 positionSAHigh 要单拎出来分析，包括这个环节的 算子的接受率，迭代过程 L+ cost 变化，最优 L 变化 等等
