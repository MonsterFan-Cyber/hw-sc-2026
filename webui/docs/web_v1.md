需求，使用 web 架构，对 docs/spec/snapshoot_format.md 当中的 .snpst 文件进行解析，并进行动画播放

## 一、页面布局要求

### 1. 整体布局结构

采用左侧绘图区域 + 右侧控制面板的布局方案：

```
┌─────────────────────────────┬──────────────────┐
│                             │  [文件选择]       │
│                             ├──────────────────┤
│                             │  播放控制         │
│      绘 图 框               │  [▶] [⏸] [⏮][⏭]│
│   (自适应缩放)              │  速度: [1x ▼]     │
│                             │  循环: [□]       │
│                             ├──────────────────┤
│                             │  帧控制           │
│                             │  开始帧: [___]    │
│                             │  ━━━━●━━━━ 45/100│
│                             ├──────────────────┤
│                             │  日志区域         │
│                             │  (可滚动)         │
└─────────────────────────────┴──────────────────┘
```

### 2. 区域划分

#### 左侧：绘图框区域
- 占据页面主体空间（约70-80%宽度）
- 自适应浏览器窗口大小
- 支持鼠标滚轮缩放、拖拽平移
- 显示所有多边形及可行域
- 帧信息显示：左下角显示"当前帧号 / 总帧数"

#### 右侧：控制面板区域
- 固定宽度（约250-300px）
- 垂直排列，分为四个模块：

**模块1：文件选择（顶部）**
- 文件选择按钮：显示当前文件名或"未选择文件"
- 支持 .snpst 扩展名过滤

**模块2：播放控制**
- 播放按钮 [▶]
- 暂停按钮 [⏸]
- 单帧后退按钮 [⏮]
- 单帧前进按钮 [⏭]
- 播放速度下拉框：0.5x、1x、2x、4x
- 循环播放复选框

**模块3：帧控制**
- 开始帧输入框：数字输入，验证范围
- 帧导航条：可拖动进度条
  - 显示当前位置
  - 拖动时实时预览
  - 标记特殊帧（存在非法多边形的帧用红色标记）
- 帧号显示：当前帧 / 总帧数

**模块4：日志区域（底部）**
- 可滚动文本区域
- 显示帧信息、合法性统计、文件元信息
- 支持复制或导出

### 3. 布局优势
- 控制面板集中，操作路径短
- 绘图框占据最大空间，视觉效果好
- 符合视频播放器常见布局习惯
- 适配移动端（控制面板可收起/展开）

## 二、文件解析要求

### 1. 二进制解析规范
- 必须严格遵循小端序读取多字节整数
- u32、f32、u8 类型必须精确对应，不能类型推断或容错
- 按文档字段顺序顺序解析，不支持随机访问

### 2. 完整性检查
- 解析时需验证文件完整性（字段数量、字节长度）
- 需处理异常情况：文件损坏、格式不符、数据截断

### 3. 数据还原
- 每帧渲染时，需计算各多边形原始包围盒中心
- 将平移向量应用到原始坐标，而非累加前一帧状态

## 三、可视化渲染要求

### 1. 多边形渲染
- 各多边形需有唯一编号标签（按多边形模块顺序，从1开始）
- 详细颜色和样式定义见"四、视觉设计方案"

### 2. 坐标系与缩放
- 自动计算所有帧的多边形包围盒，自适应缩放显示
- 支持手动缩放和平移视图（鼠标滚轮、拖拽）

### 3. 帧信息显示
- 绘图框左下角显示：当前帧号 / 总帧数
- 每帧切换时，平滑过渡（opacity 渐变 0.3s）

## 四、视觉设计方案（增强版）

### 1. 主题颜色配置

采用深色科技风（赛博朋克/数据可视化风格）：
- 主背景：#0d1117（深蓝黑）
- 控制面板背景：#161b22（深灰蓝）
- 主强调色：#58a6ff（明亮蓝）
- 次强调色：#8b949e（灰色文字）
- 边框色：#30363d（深灰边框）
- 网格背景线：#1e293b（淡色网格）

### 2. 多边形配色方案

采用渐变填充 + 发光效果：
- 每个多边形使用不同色相，饱和度 60%，亮度 50%
- 填充：半透明渐变（opacity 0.3-0.5）
- 边框：2px 实线，颜色同色相但亮度更高
- 发光效果：box-shadow: 0 0 8px ${color}

示例配色（按多边形编号循环）：
- 多边形1：#3b82f6（蓝）
- 多边形2：#8b5cf6（紫）
- 多边形3：#06b6d4（青）
- 多边形4：#10b981（绿）
- 多边形5：#f59e0b（橙）
- 多边形6+：继续循环上述颜色

### 3. 合法/非法状态配色

**合法多边形：**
- 正常显示多边形基础颜色
- 边框：柔和发光效果
- 发光：box-shadow: 0 0 8px ${color}

**非法多边形：**
- 填充：保持原色但叠加红色半透明层（opacity 0.3）
- 边框：红色发光轮廓 #ef4444
- 发光效果：双层红色发光
  - box-shadow: 0 0 12px #ef4444, 0 0 24px #ef4444
- 脉冲闪烁动画（1s 周期）

**可行域：**
- 无填充
- 边框：#6366f1（靛蓝）虚线，2px
- 发光：box-shadow: 0 0 6px #6366f1

### 4. 动态视觉效果

**帧切换动画：**
- 平滑过渡，opacity 渐变 0.3s

**非法帧标记：**
- 进度条上红色发光点
- 脉冲动画提示

**交互效果：**
- 鼠标悬停多边形：发光增强（box-shadow 扩大至 12px）
- 悬停时显示 tooltip（多边形编号、顶点数、合法性）
- 鼠标悬停按钮：颜色加深 + 微放大（scale 1.05）

**编号标签：**
- 显示位置：多边形中心
- 文字样式：白色文字，带深色阴影（text-shadow）
- 字体：等宽字体，便于对齐

### 5. 控制面板视觉

**按钮样式：**
- 默认：深灰背景 + 蓝色边框
- 悬停：背景变亮 + 发光效果
- 点击：缩放反馈（scale 0.95）

**进度条样式：**
- 轨道：深灰色
- 已播放：蓝色渐变
- 拖动手柄：白色圆形 + 发光

**日志区域：**
- 背景：半透明深色
- 文字：等宽字体，白色
- 滚动条：自定义样式（细条 + 圆角）

## 五、动画控制增强

### 1. 播放控制
- 播放速度调节（0.5x、1x、2x、4x 档位）
- 单帧步进按钮（前进一帧、后退一帧）
- 循环播放开关

### 2. 帧导航条
- 进度条显示当前帧位置
- 拖动时实时预览对应帧
- 标记特殊帧（如存在非法多边形的帧用红色标记）

### 3. 开始帧选择
- 输入框验证：必须在有效帧范围内（1 ≤ 帧号 ≤ 总帧数）
- 设置后自动跳转到该帧并暂停

## 六、日志区域要求

### 1. 帧信息日志
- 显示当前帧的合法性统计：合法数 / 总数
- 列出该帧所有非法多边形的编号

### 2. 求解日志
- 若 .snpst 文件未来扩展包含求解日志文本，需支持显示
- 当前版本可显示文件元信息：多边形数量、总帧数、可行域顶点数

### 3. 日志格式
- 简洁文本格式，按时间戳或帧号排序
- 支持复制或导出日志

## 七、交互体验要求

### 1. 文件选择
- 支持 .snpst 扩展名过滤
- 文件加载后自动解析并显示第一帧

### 2. 状态提示
- 加载中、解析中、播放中状态提示
- 解析失败时明确提示错误原因

### 3. 布局响应
- 绘图框自适应浏览器窗口大小
- 控制按钮区域固定宽度，绘图框占据剩余空间
